# 复刻招新网站学习心得
记录人：黄语馨
 ## 核心学习收获
 ### 布局与相应样式设计
 1. grid与flex的尝试使用：我在不同部分尝试使用两种布局方式，并成功将文字与背景图像排版成样例的样式

 2. 渐变与阴影的熟练应用尝试：

    - 渐变与阴影

    |                                                              |                                |
    | ------------------------------------------------------------ | ------------------------------ |
    | box-shadow: 4px 4px 4px 4px rgba(0, 0, 0, 0.1);              | 设置边框的阴影，增加立体真实感 |
    | background: linear-gradient(-135deg, transparent 50%, #fff 50%, #fff 70%, #fcd063 60%,#000 70%, #fcd063 80%); | 设置渐变的背景色               |

    - cubic-bezier动画曲线：学习到如何通过调整该参数，成功实现蝴蝶等图标的自然回弹效果，是一个很有用的功能

3. 交互体验优化

   - 悬停效果的精细化调整设计:

     .Butterfly{

     transform:traslateY(-8px) scale(1.02)

     }

     我通过这样的css设计，能够使得悬停效果出现放大缩小等效果，实现了更精细的变化效果

     注：若发现被覆盖，可通过z-index调整层级

     

### 问题解决能力

1. 核查代码的基本语法与拼写：虽然这个点非常的小，但是却是非常容易忽略并且出现问题。我在写Aboutus部分时，就曾因为把class拼写错误，导致css没有成功设置，反复调试。因此，在debug时一定要注意最简单的小细节。
2. 页面布局时的安排：在写html时一定要注意层层包裹的准则，做到层层包裹且有逻辑性。本次页面内容比我在上一阶段所写的网页详实得多，也使得html布局排版要比上一阶段讲究很多，比如属性名的设置既要能显示其内涵又要有层层包裹的关系，编写的时候就很需要做到**先思考再排版**，做到**心中有大致的盒子数量和位置安排**，具有整体意识。

3. ai工具辅助：在本次编写中，我遇到很多非常新的功能，在实现这些功能的时候，我一开始采用查询浏览器具体实现方法和具体属性的形式往下推进，但是在最后部分和导航栏部分中需要用的js的一些功能时，我发现仅靠描述没有办法搜索到我所需要的功能，于是我开始尝试用ai辅助。但是使用ai时，更**需要把问题需求描述得明确和清晰，并且需要给予反馈和调整，需要不断地尝试和调节参数**，最后才能达到想要的效果。

### 不足与缺陷

1. 整体复刻程度较低，没有完整做到所有功能
2. 在设计动态交互效果时，比较依赖css的功能，对js的技术探索较少，导致一些功能未能较好完成，后期需要对js进行更深入的学习和了解
3. 整体页面效果的流畅度不够好，并且没有给到每个板块足够的灵活性，比如参数设置比较生硬，全凭感觉而非通过盒子与盒子间准确的比例关系完成排版

### 设计感悟

通过这个项目，我深刻理解了“细节决定成败”：

- **1px的边框差异**可能影响整体的精致感
- **0.1s的动画时长调整**显著改善具体观感
- **适宜的颜色对比度和透明度**影响整体动画效果的专业感

网页的设计不仅仅是视觉效果，更是解决问题和思维过程的具体体现，每一个样式的选择、每一个交互设计，都应该做到心中有数，言之有物，希望下一次的网页开发，能够拥有更高的熟练度和对技巧的把握度。
