# 页面编后感

## 记录人：黄语馨

### 关于My-web

1. 本次网页编写，我试图通过前端三部曲，搭建一个有导航栏，并且可以通过点击导航栏内容，可以跳转到下方内容对应部分的网页。这个网页被用来做为我的自我介绍网页。但是，不得不承认，浅显了解还不足以支撑我完全的实现我的构想，其中还存在以下一些问题。例如:

   1. 跳转位置不准确:由于我设置导航栏锁定在页面最上端，导致跳转到目标位置时，标题会被遮盖，影响美观

   1. 图片插入不熟练:该问题存在两个方面。首先，在我设置的＂精彩瞬间＂部分的盒子中，插入的图片不够整齐美观，图片参数调整不美观。其次，我本想在＂首页＂部分插入背景图，但是不管是在＂主横幅＂盒子还是在＂自我介绍＂的盒子中插入，都无法达到我想要的填充效果。最终暂时留白。

   1. 搭建网页逻辑缺乏条理:我总在写HTML和css时临时补充本该写在相对靠前部分的内容，总是要写到后文，才意识到前面需要添加内容

2. 当然，十天的学习不可能毫无收获，于是，我想分享我学到的比较有趣的内容：

   1. 背景渐变：

      **linear-gradient（135deg，#07f7ff 0%，#87ceeb 50%，#1e90ff 100%）**

      我的css中有这么一条声明，左至右依次表示，渐变角度，起始颜色，中间颜色，最终颜色，各自在位置的百分比，最终达到背景渐变的效果

   2. 边框设置：

   **border-radius：25px**

   我通过这条属性，在导航栏中把边框设置成胶囊形状，成功增加了页面的趣味性，是我非常喜欢的一种页面变化。在没设置具体角度时，他是比较规整的90°，通过赋值，让他变成了一个比较圆润的形状。并且其变化十分灵活，我通过查询Google，得到

   | 所给值                                     | 对应变化                                                     |
   | ------------------------------------------ | :----------------------------------------------------------- |
   | 一个值：border-radius: 20px                | 所有四个角都是 20px                                          |
   | 两个值：border-radius: 10px 30px           | 第一个值控制 左上角 和 右下角； 第二个值控制 右上角 和 左下角。 |
   | 三个值：border-radius: 10px 30px 20px      | 第一个值控制 左上角；第二个值控制 右上角 和 左下角；第三个值控制 右下角。 |
   | 四个值：border-radius: 10px 20px 30px 40px | 四个值分别对应 左上角、右上角、右下角、左下角（顺时针顺序）。 |

   

3. 悬停时改变状态

   **.nav-link:hover**{

      background: #ff6b6b;

      color: white;

   ​    box-shadow: 0 5px 15px rgba(225,107,107,0.3);

   }

   **.hobby:hover**{

​      	background-color: #FFD700;

​      	transform: translateY(-5px);

​      	box-shadow: 0 10px 25px rgba(225,215,0,0.3);

​    	}

​	这是我在视频学习中了解到的一个JS事件，即鼠标悬停，通过这个属性，我实现了一个简	单的动态效果，增加了网页的趣味性，也是我十分喜欢的一个小设计

4. 平滑滚动效果：

​	**<a href=#"id”>**

​	这个功能是我在构思网页时想要达到的效果，但是视频课中没有具体介绍，于是我向学长咨询后，做出来的动态效果。在导航栏部分的HTML中加入这个指令，就可以通过点击导航栏中的具体板块名称，让页面自动跳转到下方对应板块，增加了页面的互动性，也是我很喜欢的一个动态效果。



### 关于“我的难题”

关于我的项目从学习了解基本框架，到构想一个网页基本样式和功能，最后成功落地，我认为，最难的步骤是将知识点真正迁移成所需要做出来的一条条代码。

我只花了五天就把一个简单版的前端三部曲介绍视频看完了，但是在开始尝试用视频学到的内容和笔记来构建我的网页时，我却对着电脑半天写不下一个字。于是我开始思考，一个网站具体的落地思路是什么，我选择直接去询问deepseek，得出了构建网页的基本步骤，项目分析——HTML规划骨架结构——css设计样式布局——添加动态功能和具体效果。我再从这样一个简单思路上开始慢慢设想我的网页所需要具备的具体内容，这样的路径走下来，果然发现填写内容的路顺畅了许多。

然而，新的问题也出现了，样式布局涉及会涉及到什么内容？想像的动态效果要怎么达成？有了上一个问题的前车之鉴，我选择不再对电脑发呆，而是转向询问有较丰富经验的学长。从询问和不停地沟通中，我不仅了解到一些比较常用的属性，也学会在“菜鸟编程”和Google中自主查询想要的属性，慢慢地搭建了我自己的网页。与此同时，我将我比较喜欢的页面交给deepseek拆解分析各网页吸引我的功能是如何达成的，慢慢在我原有的基础上增添比较简单且现阶段我可实现的动态效果，例如导航栏跳转和鼠标悬停发生的变换。

最后，看到学长学姐提问的最后一个问题，我认为我的网页当然是存在很多需要优化和填补的内容，它离我想像中的最终样貌差距还是很大，而我如果真的成为想象中和学长学姐一样的dalao，我一定会将它细细完善。十天的努力，虽然不足以让它完美，但它最终的样子已经很深地刻在我的脑海里，我希望有一天我能真的把想象中的它做出来。
