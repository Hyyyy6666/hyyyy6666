# GitHub入门学习笔记

## ***记录人：黄语馨***
## ***摘要：简单记录了Geek小组第一阶段考核学习内容，本文内容顺序按学习时间顺序记录***

### GigHub概念学习
1. 学习路径：从[廖雪峰的官网](https://liaoxuefeng.com/books/git/introduction/index.html)中简单学习了Git和GitHub的概念，
2. 学习内容：
    ①定义：**GitHub**是基于**Git**建立的一个是基于Git建立的代码托管平台。
    ②作用：**GitHub**可以看做类似一个共享的云盘，可以上传、收藏且共同修改内容，实现多方合作。

### GitHub仓库创建
1. 学习路径：从【廖雪峰的官网】(https://liaoxuefeng.com/books/git/introduction/index.html)
2. 学习内容：如何在个人GitHub里创建个人的仓库，并且与VS code同步，可以做到在VS code中修改文档并push到GitHub上
   

### readme编写
1. 学习路径：浏览他人编写的readme和GitHub自带的写作提示。
2. 学习内容：学会用emoji和markdown简单格式写简单readme。并且学会如何在readme中做笔记的索引，方便查阅想查看的笔记。

### 协作开发与代码审查的一些简单功能
1. Fork：把认为别人觉得不错的项目"复制"到自己的账户下，成为自己的独立仓库。可类比成游戏中将别人玩过的存档"复制"到自己的游戏读档目录中，再自己玩。
2. Pull Request：在Fork他人项目后，你有一个不错的idea，想要分享上传到原作者的原内容上，就可以向对方发送一个"Pull Request"，对方自己考虑后决定时候要增加你修改的部分。
