# 关于git的学习感想
## 感想
- 我学习了Git的基本命令，如 `git init`、`git add`、`git commit`、`git push` 等操作，逐渐掌握了从本地仓库到远程仓库的完整提交流程。通过多次练习，我发现命令行操作能更清楚地了解Git的内部机制，比如分支管理、版本回滚、冲突解决等。

- Git不仅是一种工具，更是一种思维方式。它让我养成了良好的代码管理习惯。让每个开发者都能在本地进行独立开发，并在需要时合并到远程仓库，极大提高了团队协作效率。

- 学习 Git 的过程，很像学习一门乐器或一门运动。起初是枯燥和困难的，但一旦你突破了那个瓶颈，掌握了它的“语言”和“节奏”，它就会变成你能力的自然延伸，为你打开一扇新世界的大门。


## 困难

1. “工作区”、“暂存区”、“本地仓库”、“远程仓库”概念的区分
2. 是在使用Git Bash时，记忆命令比较困难
3. 第一次遇到 `merge conflict` 时，看着满屏的 `<<<<<<<`, `=======`, `>>>>>>>`，内心是崩溃的，不知道该如何下手。
