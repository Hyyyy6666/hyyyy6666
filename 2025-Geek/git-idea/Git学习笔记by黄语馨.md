### 内容1：关于在创建的仓库中add文件
1. 具体场景:我尝试将新修改文件add到初始化好的仓库中.经常出现该指令**fatal: pathspec '文件名' did not match any files。**
2. 原因分析：
-未将文件放入GIt仓库中,GIt无法调控。
-打开Bash的位置不正确,即未在正确的位置操作。
3. 解决方法：
-可使用'ls -la'的指令查看文件是否存在我所设置好的仓库中。
-可使用'cd'指向具体目录,再尝试操作。

### 内容2：关于如何提交修改
1. 具体场景:我在尝试commit文件的过程中时常发现commit不成功的情况。
2. 原因分析:暂无。
3. 解决方法:我在提交信息栏中简单加入语句,部分情况下便可快速提交成功。

### 内容3：同步远程仓库
1. 具体场景:我时常在同步远程仓库时会遇到提示,告诉我双方有冲突；我的GitHub上所有文件会突然出现重合或者乱码的内容。
2. 原因分析:我认为应该是由于我偶尔会因为觉得在VScode中push文件麻烦,所以会直接在GitHub中修改我的具体内容,导致两端文件存在同时被修改的情况,且修改部分,时常会重叠或冲突。
3. 解决方法:直接在VScode中合并或者删除，选择具体留下的内容，再进行上传。

### 内容4：markdown编辑层级混乱
1. 具体场景:在VScode中编写markdown文档的时候层级清晰,但是最终呈现效果混乱。
2. 原因分析:markdown解析工具对代码缩进的解析规则不同。
3. 解决方法:使用列表符号(如-/1.)等其他分层规范。
