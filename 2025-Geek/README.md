## 1️⃣阶段一 【Geek 的出厂设置 - `GitHub` 入门与 Markdown 学习】

### 阶段时长：`1.5` 天

### 🛒学习内容

1. 初步了解 `GitHub` 

2. 学习使用 Markdown

   > Markdown 可能是未来 Geek 们接触最多的工具，当你真正爱上它的时候，电脑中的 Word 似乎再也没怎么打开过......

### 📑任务内容

❗**注意：考核期所有笔记/文档须为 `.md` 格式，当前与此后的所有任务内容均上传至你在 `GitHub `创建的名为 Tasks 的仓库**

1. 炫炫技！ 请你尽可能用上你学的 Markdown 语法技巧，组织一个整洁规范的文档  跟我们说说：
   - 既然你选择了 Geek 组，那么你觉得，什么是 Geek 精神？
   - 你认为 `Github` 是干啥的？里面那些`Pull request`，`Fork`之类的词语又是干啥的？这些你又是通过哪些途径学到的？ 
   - 有学长一开始了解 `Github` 时，光是进网站就花了半天，你遇到过这种情况吗？如果遇到过，你又是如何解决的？如果没遇到过，那你觉得学长的情况是因为什么？
   - 最后，说说你的这份文档有什么理由称得上整洁规范？你又遵循了哪些规范？
   
2. 创建 Profile README 仓库, 美化你的 GitHub 主页

   - 使用 Markdown 语法编写个人介绍

   - 探索并学习优秀模板
     **◦  参考至少 2 个优秀的 GitHub Profile README(并给出对应的参考链接🔗)**
     ◦ 记录📝在你的学习笔记中记录你参考了哪些案例，学到了什么技巧

3. **[选做-进阶]** 了解 Markdown 的原理，探索一下你的编辑器还有什么奇妙或者方便的功能，写成文档分享给大家和未来的学弟学妹吧。

* **[可选] Markdown 进阶笔记**

## 3️⃣阶段三 【 Web 相关领域的基础 - HTML / CSS / JavaScript 】

### 阶段时长：`10` 天

### 🛒学习内容

❗**注意：本阶段的各个学习内容存在强联动，不建议单独学习，建议分别进行基础语法了解后联动学习。重点学习 HTML 与 CSS 。**

1. 学习 HTML 与 CSS 基础，了解 JS，并综合应用
2. 学会用 `GitHub Pages` 部署静态页面

### 📑任务目标

1. 👉使用 HTML、CSS 与 JS 制作一个属于你自己的静态网页，展示任何你想展示的内容，然后使用`Github Pages` 部署该页面。👈

- **致如此努力的你**： 坚持到现在的你非常棒！而本项目将是你第一个完整实现的前端项目，或许有许多 BUG、障碍挡路，但我们希望你用心思考，多多实践、终能披荆斩棘，用一行行代码展示出你最耀眼的光辉。

- **这一阶段很重要**：本阶段各位同学的成果会经过审核和打分，该分数不仅是重要的考核指标，得分前三的作品还将在组内公开展示，甚至纳入组内历年优秀考核作品仓库！
- **我们鼓励创新，但请力所能及**：如学有余力，可以添加独属于你的创新亮点，但我们不希望直接借助 AI 实现本不属于你的东西，也不希望你为了创新而忽视了基本。
- **我们允许简单，只要你尽力**：如果你确实时间较少或者学习相对吃力，可以只用 HTML、CSS 与 JS 的一两种制作一个简单的页面demo，尽力而为并持续学习，一定会成为优秀的人。

2. 无论你的项目如何，详细编写一篇 “编后感” ，说说：
   - 你构想了一个什么项目，最终完全实现你的构想了吗？
   - 你在项目中，是如何用 HTML 给它搭建骨骼、用 CSS 给它增添颜值、用JS给它赋予变化的？
   - 我们总是在关关难过关关过中成长，说说你在项目中遇到的最大难题是啥？你又是如何克服和解决的？
   - 你认为当前的项目还需要优化和补完吗？如果将来你成为一方 dalao，你会想回到最初为这个项目填上最后一块砖吗？

### ❤友情提示

1. 推荐：[菜鸟教程](https://www.runoob.com/)；[w3school 在线教程](https://www.w3school.com.cn/)； [廖雪峰JS教程](https://www.liaoxuefeng.com/wiki/1022910821149312/1023442583285984) ；[小破站](https://tower.im/teams/88948/documents/18264/bilibili.com)（建议学习每个之前都先去找一些视频教程来看，先从大体了解一个东西是什么，然后再去看文档，边看边 Coding 是最好了）
